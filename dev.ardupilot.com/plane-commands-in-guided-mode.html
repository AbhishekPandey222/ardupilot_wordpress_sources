This article lists the commands that are handled by Plane in GUIDED mode (for example, when writing GCS or Companion Computer apps in DroneKit). Except where explicitly stated, most of these can also be called in other modes too. [note]The list is inferred from Plane's GCS_Mavlink.cpp [/note]

Movement commands

For movement, Plane uses: MAV_CMD_NAV_WAYPOINT message encoded with the "current" parameter set to "2" to indicate that it is a guided mode "goto" message. MAV_CMD_CONDITION_CHANGE_ALT message encoded with the "current" parameter set to "3" to indicate that it is a guided mode "altitude change" message.

MAV_CMDs

These MAV_CMDs can be processed if packaged within a COMMAND_LONG message. MAV_CMD_NAV_LOITER_UNLIM MAV_CMD_NAV_RETURN_TO_LAUNCH MAV_CMD_DO_SET_ROI MAV_CMD_MISSION_START MAV_CMD_COMPONENT_ARM_DISARM MAV_CMD_DO_SET_SERVO MAV_CMD_DO_REPEAT_SERVO MAV_CMD_DO_SET_RELAY MAV_CMD_DO_REPEAT_RELAY MAV_CMD_DO_FENCE_ENABLE MAV_CMD_DO_SET_HOME MAV_CMD_START_RX_PAIR MAV_CMD_PREFLIGHT_CALIBRATION MAV_CMD_PREFLIGHT_SET_SENSOR_OFFSETS MAV_CMD_DO_SET_MODE MAV_CMD_PREFLIGHT_REBOOT_SHUTDOWN MAV_CMD_DO_LAND_START MAV_CMD_DO_GO_AROUND MAV_CMD_REQUEST_AUTOPILOT_CAPABILITIES MAV_CMD_DO_AUTOTUNE_ENABLE These MAV_CMD commands can be sent as their own message type (not inside COMMAND_LONG): MAV_CMD_DO_DIGICAM_CONFIGURE MAV_CMD_DO_DIGICAM_CONTROL MAV_CMD_DO_MOUNT_CONFIGURE MAV_CMD_DO_MOUNT_CONTROL
