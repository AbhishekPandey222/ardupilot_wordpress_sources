The following is a list of PIN variables in copter, plane and rover as of Jan 2014. Users may set any of these pins to point at any valid pin of the right type (analog or digital). More will likely be added soon as some pins are hard-coded that should have PIN variables. Most of these need to be set as "advanced parameters".

Plane

  • ARSPD_PIN (analog)
  • BATT_CURR_PIN (analog)
  • BATT_VOLT_PIN (analog)
  • RELAY_PIN (digital)
  • RELAY_PIN2 (digital)
  • RELAY_PIN3 (digital)
  • RELAY_PIN4 (digital)
  • RSSI_PIN (analog)
  • SONAR_PIN (analog)
  • SONAR_STOP_PIN (digital)

Copter

  • BATT_VOLT_PIN (analog)
  • BATT_CURR_PIN (analog)
  • RELAY_PIN (digital)
  • RELAY_PIN2 (digital)
  • RELAY_PIN3 (digital)
  • RELAY_PIN4 (digital)
  • RSSI_PIN (analog)

Rover

  • BATT_CURR_PIN (analog)
  • BATT_VOLT_PIN (analog)
  • RELAY_PIN (digital)
  • RELAY_PIN2 (digital)
  • RELAY_PIN3 (digital)
  • RELAY_PIN4 (digital)
  • RSSI_PIN (analog)
  • SONAR_PIN (analog)
  • SONAR_STOP_PIN (digital)
  • SONAR2_PIN (analog)
  • SONAR2_STOP_PIN (digital)
  • AUTO_TRIGGER_PIN (digital)

Available pins on each board

Each board has a range of possible pins that may be assigned to the above pin variables. For analog pins you need to be careful that the allowed voltage input range of the pin matches the device you are attaching. For digital output pins you need to ensure the pin can provide enough current. The digital and analog pins available on each board are documented in the associated board overview: Pixhawk Overview PX4 Overview AMP2 Overview [note]If you don't want to fiddle with the code but still want to add an arbitrary analog sensor to one of the analog pins and read it via MAVLink, the easiest way is to do that is use the built-in Sonar sensor function, even though your sensor may not actually be a sonar sensor. To do that, attach the sensor the A0 on APM 2.x and in the MAVLink param list, set SONAR_ENABLE to 1. You can then read and graph the data via MAVLink with the RANGEFINDER.voltage MAVLink message. If you're using the Rover code, you'll still use the SONAR function, but you have more flexibility over which analog pin you're attaching your sensor to and how to read that data.
