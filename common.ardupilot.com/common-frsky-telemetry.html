This article describes how to transmit vehicle information from the autopilot and display it on an FrSky transmitter ("FrSky Telemetry").

Overview

An FrSky RC transmitter will not be able to display vehicle telemetry information "out of the box" because ArduPilot normally outputs data using the MAVLink protocol, while FrSky uses its own non-MAVLink protocol. This article explains a number of approaches for getting information from the autopilot to an FrSky transmitter (including the Taranis).

Software/hardware solution

ArduPilot provides a driver that can directly output the FrSky protocol. This section explains how you can connect and configure a Pixhawk to communicate with a D-Receiver or X-Receiver. [note]This feature is only officially supported on the Pixhawk (running AC3.2 or higher) but there is a DIY solution for the APM2.x here.[/note] [note]These instructions are largely based upon information from the Pixhawk website.[/note]

D-Receiver setup

For the D-Receiver setup you will need:
  • FrSky D telemetry capable receiver/transmitter set like this one from 3DRobotics, OR
  • a FrSky D telemetry capable receiver such as the FrSky D4R-II (HobbyKing) and FrSky D telemetry capable transmitter such as the Taranis which runs the OpenTX open source transmitter software.
  • TTL-to-RS232 converter such as the FrSky FUL-1 which is sold by many retailers including these on ebay.
  • DF13 6 position connector which will be cut and soldered to the TTL-to-RS232 converter
  • Molex Picoblade 4 position telemetry wire which will be cut and soldered to the TTL-to-RS232 converter (should be included with the receiver)

X-Receiver setup

For the X-Receiver setup you will need:
  • FrSky X telemetry capable receiver such as the FrSky X4R, X6R or X8R .
  • FrSky X telemetry capable transmitter such as the Taranis which runs the OpenTX open source transmitter software.
  • TTL-to-RS232 converter such as the FrSky FUL-1 which is sold by many retailers including these on ebay.
  • SPC cable such as SPC Cable
  • DF13 6 position connector which will be cut and soldered to the TTL-to-RS232 converter
  • Molex Picoblade 4 position telemetry wire which will be cut and soldered to the TTL-to-RS232 converter (should be included with the receiver)

FrSky Telemetry setup in Mission Planner

To enable the FrSky Telemetry output on the Pixhawk's Telem2 port, please connect with the Mission Planner and then open the Config/Tuning | Full Parameter List page and set the SERIAL2_PROTOCOL parameter to "3" for D-Receiver and "4" for X-Receiver. [note]The information above is for ArduCopter 3.3. Prior to AC3.2 the values are 2 for D-Receiver and 3 for X-Receiver.[/note]

Transmitter set-up

Please refer to the OpenTX manual for how to control which values from the telemetry feed will be displayed on the transmitter's screen.

Hardware solutions

This section contains hardware-only MAVLink to FrSky converter solutions. Please feel free to add your own solutions (or ask us to).

APM MavLink to FrSky SmartPort Converter (Airborne Projects)

Airborne Project's APM MavLink to FrSky SmartPort Converter converts MAVLink messages to FrSkySmartPort format. It can directly be connected to the Taranis Radio. You only have to load the Taranis telemetry modules and configure in Mission Planner. No soldering required! [caption id="" align="alignnone" width="500"] Airborne Projects: APM MavLink to FrSky SmartPort Converter[/caption] The converter includes all needed cable. It features a hardware modified version of the Arduino Nano and be powered directly from the Taranis receiver. For more information see the QuickStart Guide (www.airborneprojects.com).
